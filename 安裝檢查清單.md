# ✅ 安裝檢查清單

使用這個清單確保所有東西都準備好了!

---

## 📋 第一步:檢查軟體安裝

### 1. 檢查 Node.js

**怎麼檢查**:
1. 按 `Win + R`
2. 輸入 `cmd` 按 Enter
3. 輸入: `node -v`
4. 應該看到類似 `v20.11.0` 的版本號

**結果**:
- ✅ 有看到版本號 → 已安裝,繼續下一步
- ❌ 出現錯誤 → 需要安裝 Node.js

**如何安裝**:
- 前往: https://nodejs.org/
- 下載左邊的「LTS」版本
- 安裝後重新啟動電腦

---

### 2. 檢查 Pandoc

**怎麼檢查**:
1. 在同一個 cmd 視窗中
2. 輸入: `pandoc --version`
3. 應該看到版本資訊

**結果**:
- ✅ 有看到版本號 → 已安裝,繼續下一步
- ❌ 出現錯誤 → 需要安裝 Pandoc

**如何安裝**:
- 前往: https://pandoc.org/installing.html
- 下載 Windows 版本
- 安裝後重新啟動電腦

---

## 📋 第二步:設定 API Key

### 1. 找到 .env 檔案

**位置**: 在專案資料夾中  
**檔名**: `.env`

**找不到?**
- 可能是隱藏檔案
- 在檔案總管中,點選「檢視」→ 勾選「隱藏的項目」

### 2. 編輯 .env 檔案

**用記事本打開**:
1. 右鍵點擊 `.env`
2. 選擇「開啟檔案」
3. 選擇「記事本」

**修改內容**:
```
原本: GEMINI_API_KEY=請把你的API金鑰貼在這裡
改成: GEMINI_API_KEY=你的實際API金鑰
```

**範例**:
```
GEMINI_API_KEY=AIzaSyABC123def456GHI789jkl
```

**儲存**: 按 `Ctrl + S`

---

## 📋 第三步:安裝程式套件

### 1. 開啟終端機

**方法一**:
1. 在專案資料夾上按右鍵
2. 選擇「在終端機中開啟」

**方法二**:
1. 按 `Win + R`
2. 輸入 `cmd`
3. 輸入: `cd d:\project\AI教材轉換專案115`

### 2. 安裝套件

**輸入指令**:
```
npm install
```

**等待完成**:
- 會看到很多文字跑過
- 等待 3-5 分鐘
- 看到「added XXX packages」就完成了

**結果**:
- ✅ 成功 → 繼續下一步
- ❌ 失敗 → 檢查網路連線,重試一次

---

## 📋 第四步:測試程式

### 1. 啟動程式

**最簡單的方法**:
- 雙擊 `啟動程式.bat`

**或是用指令**:
```
npm start
```

### 2. 檢查是否成功

**應該看到**:
```
========================================
  🚀 AI 教材轉換系統已啟動!
========================================

  📍 網址: http://localhost:3000
```

**瀏覽器會自動打開**:
- 看到紫色漸層的上傳頁面 → ✅ 成功!
- 沒有打開 → 手動開啟瀏覽器,輸入 `http://localhost:3000`

---

## 📋 第五步:測試轉換功能

### 1. 上傳測試檔案

**使用提供的範例**:
- 檔案名稱: `測試教材範例.md`
- 位置: 專案資料夾中

**操作步驟**:
1. 點擊上傳區域
2. 選擇 `測試教材範例.md`
3. 點擊「開始轉換」
4. 等待進度條跑到 100%

### 2. 下載檔案

**應該會看到**:
- 📑 下載 PDF 檔案
- 📝 下載 Word 檔案

**點擊下載**:
- 檢查 PDF 是否能正常開啟
- 檢查 Word 是否能正常開啟

### 3. 檢查品質

**PDF 應該有**:
- ✅ 清楚的標題(有顏色)
- ✅ 程式碼區塊(深色背景)
- ✅ 表格(有斑馬紋)
- ✅ 頁碼(在底部)

**Word 應該有**:
- ✅ 標題格式
- ✅ 程式碼區塊
- ✅ 表格
- ⚠️ 目錄頁碼可能是空的(這是正常的,需要手動更新或安裝 LibreOffice)

## 📋 第六步:回報結果

### 成功 ✅

**如果一切正常**:
- 恭喜!基礎版本已經可以使用了!
- 可以開始轉換您自己的教材
- 想要更多功能,請看「implementation_plan.md」

### 遇到問題 ❌

**常見問題**:

**問題 1**: 程式無法啟動
- 檢查 Node.js 是否安裝
- 檢查是否執行過 `npm install`

**問題 2**: 轉換失敗
- 檢查 API Key 是否正確設定
- 檢查網路連線
- 檢查檔案是否為 .md 格式

**問題 3**: Word 無法產生
- 檢查 Pandoc 是否安裝
- 重新安裝 Pandoc

**問題 4**: PDF 樣式不正確
- 這可能是正常的,基礎版樣式較簡單
- 可以在第二階段加入更專業的樣式
